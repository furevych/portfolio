State Financial Overview: Income vs. Major Expenses

Project Purpose: This project analyzes and visualizes the relationship between household incomes and significant expenses across U.S. states. By normalizing and scoring these financial metrics, the analysis aims to provide insights into the economic conditions in different states, facilitating comparisons that reveal differences in living costs and income levels.

The project involved processing and aggregating a complex datasets using Python (matplotlib, pandas), with a focus on visualizing the balance between incomes and significant expenses. Flexible functions were created for cleaning, merging, and normalizing data. These functions make it easy to handle current data and integrate future updates.

Methodology and Calculations:

This analysis is based on actual values from reputable, publicly available datasets. To facilitate comparison and visualization, some prices levels and values were transformed into scores and ratios.

  • Data Transformation: Real data for household income, housing costs, medical expenses, gas prices, taxes, and cost of living (CPI) were rounded and normalized. Expenses were scored from 1 to 10, while incomes, reflecting a broader range, were scored from 1 to 100. Housing Comparison: Actual prices and their ratios were used to compare incomes and evaluate housing affordability.
  • Grading System: The normalized scores provide a simplified comparison of how different states rank in terms of income and costs. Higher scores indicate higher values for that category.
  • Proportional Representation: Since all values originated from actual dollar amounts, the scores effectively represent relative differences in costs and income across states. Equal grading conditions were applied to all states to ensure consistency.

    • The graphical visualizations offer insights into the proportions between incomes, expenses, and housing affordability, with adaptable Python code enabling other various data manipulations.

Flexible Functions for Data Aggregation and Future Integration:

The code is structured with adaptable components to handle various datasets, allowing updates and new data sources to be integrated easily. Key functions ensure data matching, cleaning, and merging for streamlined analysis and visualization.

  • State Name Matching: Standardization of state names is achieved through regex, which removes extra spaces and normalizes case. This ensures accurate matching across different datasets, preventing mismatches due to formatting differences.
  • Data Cleaning: String values, including percentages and commas, are converted into numeric values, making all expense and income data usable for calculations and visualizations. This process can be applied to future datasets for consistent standardization.
  • Data Loading and Preprocessing: Selected columns are imported from CSV files, allowing for the addition of new datasets by specifying file paths and columns. The data is merged by the 'State' column, ensuring that all states are included, even if some categories are not present. Large, complex dataset with subcolumns is pre-processed in a separate script and then combined into a unified file.
  • Main Processing: The entire workflow is streamlined to load, clean, and merge datasets, allowing for straightforward updates by simply adding new data sources. The merged data is saved for further analysis, making the system adaptable to ongoing data updates.
  • Visualization: The aggregated data is visualized clearly, comparing normalized expenses against income. This function is designed for flexibility, enabling users to visualize any state's data with minimal input, ensuring adaptability for future visualization needs.
